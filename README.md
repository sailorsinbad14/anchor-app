# 锚 · Anchor

> 焦虑应急工具 · 单文件 HTML

一个用来在焦虑来袭时打开的小工具。不是治愈焦虑的工具,是给焦虑一个不用刷手机的出口。

[Demo 截图(8 张)](#screenshots)

---

## 它是什么

焦虑的浪有自己的生命周期。一个不被行动喂养的高峰,通常 90 秒到几分钟就开始衰减。难的不是熬过去,是什么都不做。

反复刷手机是焦虑的"行动喂养"——焦虑→刷手机→暂时缓解→焦虑加剧→再刷,这个回路被强化得越多,下次焦虑来得越快、越猛。

「锚」做的事情很简单:在你正要开始刷手机的那一刻,给你提供一个替代路径。

---

## 功能

四个入口,不复杂:

- **浪来了** — 应急四步流程:4-2-8 呼吸 / 离机 45 分钟 / 动起来 / 写下来
- **今日 3 行** — 每天两分钟的觉察练习,记录"出于真实"和"出于表演/防御"的事
- **记录这次焦虑** — 触发、身体感受、念头、冲动、实际行为五个维度的结构化记录
- **回看** — 翻看过去的焦虑记录和反思,看见自己的模式

工具里包含两个内嵌组件:

- **呼吸引导** — 跟着圆圈节奏 4-2-8 呼吸的可视化引导
- **离机倒计时** — 默认 45 分钟,撑不住可以提前结束,不强制

---

## 技术

- 单文件 HTML,约 900 行
- 纯前端:HTML + CSS + 原生 JS,无任何依赖、无构建步骤
- 数据存储:`localStorage`,所有内容只在用户浏览器里
- 提示音:Web Audio API 现场合成,不引外部音频
- 无埋点、无分析、无第三方脚本

## 运行

```bash
git clone https://github.com/sailorsinbad14/anchor-app.git
cd anchor-app
# 直接用浏览器打开 index.html 即可
open index.html        # macOS
xdg-open index.html    # Linux
start index.html       # Windows
```

或者更推荐:

把 `index.html` 上传到任何一个静态文件服务(GitHub Pages、Vercel、Netlify、自己的 VPS 都行),然后在手机浏览器里把页面**添加到主屏幕**。这样你就有了一个像 App 一样的入口图标,但不占应用列表、不申请任何权限、不联网。

## 部署到 GitHub Pages

1. Settings → Pages → Source 选 `main` 分支、`/ (root)` 目录
2. 等几分钟,访问 `https://sailorsinbad14.github.io/anchor-app/`

## 使用建议

不建议每天打开它。它不是社交产品,也不是日记本。它是消防栓——平时不需要,焦虑来了再打开。

「今日 3 行」是个例外,可以当成日常的两分钟练习。问自己:今天哪件事是出于真实?哪件事是出于表演?哪一刻想换掉?这三个问题坚持问一段时间,行为会自己开始变。

## 文案与设计说明

文案部分参考了认知行为疗法(CBT)和接纳承诺疗法(ACT)的概念。比如「想做什么」和「实际做什么」之间的距离这种表述,来自 ACT 关于"价值-行为"的讨论。

但工具刻意避开了"治愈""疗愈""自爱""能量"这类容易煽情的词。焦虑的人不需要被安慰,需要的是结构。

设计上沿用深色 + 单一暖金强调色,避开蓝色(医疗冷感)和粉色(情绪化)。整体节奏是慢的——按钮反馈 0.2s 渐变、呼吸引导默认 5 轮、离机倒计时默认 45 分钟。所有数字都是可调的,在源码里搜变量名修改即可。

文案底稿是我自己写的,Claude 帮我反复打磨表达。
